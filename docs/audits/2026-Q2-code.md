## Notes

- The E501s are concentrated in long string literals inside dict definitions. Two valid fixes:
  - (a) raise the line ceiling for those files via a `pyproject.toml` per-file-ignore (`"app/seeds/tasks/*.py" = ["E501"]`)
  - (b) restructure the long lines using triple-quoted strings or breaks at `{var}` boundaries
- (a) is a 2-line config change; (b) is mechanical reformatting. Recommend (a) for seed/catalog content files (where readability of the natural-text strings matters more than the column count).
